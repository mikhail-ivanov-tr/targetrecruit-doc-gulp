    ready :Boolean
    sfdx-source/shared/main/default/lwc/customElement/customElement.js, line 289

    This flag is used to specify an initial loading state. By default it equals to false. Once initial call to an Apex method from connectedCallback() is successfully done via CustomElement#apex it is set to true. If during the Apex method call some exception happened and caught/registered by CustomElement#addError then this flag will be left as false. Use this flag to identify whether to show functional content of the component once the initial Apex method call is successfully done (ready = true).

    Example

    It shows the usage of all capabilities of CustomElement. There are 2 LWC components which extend CustomElement: Parent component and Child component. Both components have 2 buttons. First button is disabled if the component is making a call to Apex via CustomElement#apex. Second button is disabled if any of the components are making a call to Apex via CustomElement#apex. So, if user clicks any button on Parent component then 2 buttons on Parent and 'Global' button on Child will be disabled. If user clicks any button on Child component then 2 buttons on Child and 'Global' button on Parent will be disabled.

    Messages:

    Parent component contains 2 <c-messages> LWCs. The first (type="alert") is to show the messages of the current (parent) LWC only. See CustomElement#messages. The second LWC (type="toast") is to show all messages which will happen in the both parent and child components. See CustomElement#messagesGlobal.

    Initial loading:

    When initial rendering happens and some database data is required use CustomElement#apex inside of connectedCallback() to load it from Apex code.

    Use <lightning-spinner> with CustomElement#initialLoading as a rendering conditions to show the spinner before the functional content will be rendered and use <div> with CustomElement#ready to show the functional content.

    You can also add a block which uses CustomElement#initialError, CustomElement#initialErrorLabel and CustomElement#closeWindow to show a Back/Close button in case some exception occurred and processed by CustomElement#addError during connectedCallback() phase.

    Apex calls:

    There are 2 additional properties/flags to identify whether the Apex call in progress or not. Use CustomElement#loading to identify if the current component is doing the Apex call using CustomElement#apex. Use CustomElement#loadingGlobal to identify if any component is doing the Apex call using CustomElement#apex.

    parent.html

    <template>
	<!-- Shows current component's messages. -->
	<c-messages messages={messages} type="alert"></c-messages>

	<!-- Shows messages of all CustomElement components. -->
	<c-messages messages={messagesGlobal} type="toast"></c-messages>

	<!-- Shows a spinner till initial call to Apex code is completed. -->
	<lightning-spinner size="large" variant="brand" if:true={initialLoading}></lightning-spinner>

	<!-- Shows a Back/Close button in case an error occurred. -->
	<div if:true={initialError}>
		<lightning-button onclick={closeWindow} label={initialErrorLabel}></lightning-button>
	</div>

	<!-- Functional content is inside of the div
		and its appearance is controlled by CustomElement#ready -->
	<div if:true={ready}>
		<!-- The button is disabled when CustomElement#apex call
			of the current component is in progress. -->
		<lightning-button label="Local" disabled={loading}
				onclick={handleButtonClicked}></lightning-button>

		<!-- The button is disabled when CustomElement#apex call
			of any CustomElement component is in progress. -->
		<lightning-button label="Global" disabled={loadingGlobal}
				onclick={handleButtonClicked}></lightning-button>

		<c-test-cache-child></c-test-cache-child>
	</div>
</template>

    parent.js

    import CustomElement from "c/customElement";

import getAccounts from '@salesforce/apex/testCacheController.getAccounts';
import getAccountsWithApexInfoMessage
	from '@salesforce/apex/testCacheController.getAccountsWithApexInfoMessage';

export default class Parent extends CustomElement {
	async connectedCallback() {
		try {
			// getAccounts Apex method returns Account[],
			// so this is what will be assigned to the "const accounts" JS variable.
			const accounts = await this.apex(getAccounts);

			// Adds a value of accounts.length as a warning message
			// to CustomElement#messages and CustomElement#messagesGlobal.
			this.addWarning(accounts.length);
		} catch (e) {
			this.addError(e);
		}
	}

	// This callback will be called every time the component's DOM is changed.
	renderedCallback() {
		// You need the following statement only
		// if you override renderedOnceCallback() also.
		super.renderedCallback();

		// This log message will be added to the console
		// multiple times on the load of the component.
		console.log(`Component rendered.`);
	}

	// This is a new CustomElement callback which will be called
	// only one time when the component initially rendered.
	renderedOnceCallback() {
		// This log message will be added to the console
		// just one time on the load of the component.
		console.log(`Component rendered first time.`);
	}

	// No need for try/catch here as any "handle..." function of the class
	// which extends CustomElement will be wrapped by the try/catch automatically.
	async handleButtonClicked() {
		// getAccountsWithApexInfoMessage Apex method returns Logger.AuraResult type,
		// so "const accounts" JS variable will be assigned
		// with the Logger.AuraResult.result value
		// and the Logger.AuraResult.messages will be added automatically
		// to CustomElement#messages and CustomElement#messagesGlobal.
		const accounts = await this.apex(getAccountsWithApexInfoMessage);

		// Adds a value of accounts.length as a success message
		// to CustomElement#messages and CustomElement#messagesGlobal.
		this.addSuccess(accounts.length);
	}
}

    closeWindow(delay)
    sfdx-source/shared/main/default/lwc/customElement/customElement.js, line 711

    The function redirects to the previous page of the current window/tab (if window.history has previous URLs, usually it is true if the component is opened as a page in the same browser's tab or window) or closes current window/tab (otherwise, usually it is true if the component is opened as a page in the new browser's tab or window). If the current component is inside an IFRAME and window.history doesn't have the previous URL then the parent window/tab of the IFRAME will be closed.

    Parameters:
    Name Type Description
    delay Number

    If the attribute is set then the window/tab will be redirected back or closed after the milliseconds provided by the attribute. Otherwise, the action will happen immediately.

    async dispatchAsync()
    sfdx-source/shared/main/default/lwc/customElement/customElement.js, line 785

    Allows to send events, wait till async handlers completed and capable to cancel the events from async handlers.

    Properties:
    Name Type Description
    detail object
    Properties
    Name Type Description
    asyncHandlers Array.<Promise>

    An empty list of Promises. If a listener of the event has some asynchronous process then the listener code body must be wrapped with a Promise and pushed to the asyncHandlers list. If the Promise's resolve function will be called with true value of the parameter then it will be considered as success. Otherwise, the event will be canceled.

    Example of async listener.

    handleFilesUpload(event) {
	event.detail.asyncHandlers.push(new Promise(resolve => {
		setTimeout(() => {
			// Approves the upload if the uploaded file's name is `valid.txt`.
			// Otherwise, cancels the event.
			resolve(event.detail.files[0].name === `valid.txt`);
		}, 1000);
	}));
}
